（读书笔记）深入浅出React（一）：React的设计哲学

2017-07-05

上一篇文章介绍了React的目的，就是开发数据不断变化的大型应用程序，而这其中的关键就是虚拟DOM，而JSX就是一种利用HTML语法来构建虚拟DOM的语言。

使用JSX定义直观的用户界面

JSX是将HTML直接嵌入到JavaScript中的一种XML语法，抛开了传统的将业务和逻辑相分离的“最佳实践”。

它的语法很像HTML，但并不是HTML。之后JSX通过翻译器转换到纯JavaScript再由浏览器执行。

可以将JSX理解为一个比较高级但很直观的语法糖。

它使用了js自带的语法和特性，而不需要使用模板引擎，使开发人员只需要利用代码去构建整个界面。

JSX中的大花括号中是JavaScript语法。

在JSX中使用样式，属性值不能为字符串而必须为对象。所以JSX中的样式都用两个大花括号括起来，第一个大花括号是标准JSX语法，代表这其中是JavaScript语法，第二个大花括号代表这是JavaScript的对象表达式。

所谓组件，其实就是状态机器

React将界面看做简单的状态机器。当组件处于某个状态时，那么就输出这个状态对应的用户界面

React的组件与外界交互的除了状态（state）外，还有就是属性（props）。属性一般被认为是只读的，在组件初始化的时候传入，是不可修改的。而状态是用来更新组件UI的，一般由组件内部去维护。

组件很少需要对外公开方法，唯一的交互途径就是props。

每个子节点都是通过父节点的render方法去构造的。

每个组件都会有一个render方法，这个方法返回组件的实例。最终整个界面得到一棵虚拟DOM树。

每一次界面变化都是整体刷新

React通过框架自身解决了局部UI需要更新的问题。实现途径就是通过虚拟DOM。

而局部UI更新最重要的就是React的Diff算法。

Diff算法基于两种假设：

  1. 不同类型的组件产生不同的DOM结构。
  2. 对于同一层次的一组子节点，它们可以通过唯一的id进行区分。

单向数据流动Flux

Flux定义了一种单向数据流的方式，来实现View和Model之间的数据流动。

Flux是一种单向数据流和单向数据绑定的思想，用来定义组件之间、组件和数据模型之间如何通信。

Redux、Reflux、Flummox都属于Flux的第三方实现。

双向数据绑定是指Model可以引起View的变化。View也可以接受用户输入引起Model的变化。

Flux中的Store是存储应用程序的状态。

对于一个小组件，状态可以在内部进行维护。

而对于多个组件构成的应用程序，某些需要组件之间共享的状态要放到Store中进行维护。

每当Store发生变化时，View也会进行相应更新。

Flux引入了Dispatcher和Action的概念，Dispatcher是一个全局的分发器负责接收Action，而Action可以来自于用户的某个界面操作，也可以来自于服务器端的某个数据更新。

只读数据模型：Immutability

React提倡利用只读数据来建立数据模型。所有数据都是只读的，如果需要修改某个数据，只能产生一份新的数据。
